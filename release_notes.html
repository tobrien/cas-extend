Release Notes

Release 2.2 - (April 30, 2006)

This release adds a single parameter that is required to support application servers that are behind other servers responsible for SSL. There are situations when a servlet container is neither resposible for SSL, nor is it aware that a request is secure because the SSL has been off-loaded to a load balancer or a web-server that is not passing on SSL information to the servlet container. In these sutations, even though a request for a protected service may have been made with an https scheme, the CAS Filter does not know that and it crafts a serviceURL with an http scheme. With this property you can force the server to use a certain scheme.

  • Added serviceScheme parameter, forces the scheme of the service URL when CAS filter computes serviceUrl off of casServerName. For more information, see the configuration documentation.

Release 2.1.1 - (late February 2006)

This is the initial fork of the Yale CAS Client codebase. In this release, we've added support for a number of options that go above and beyond the initial feature set. Please see the documentation for this filter, it is not secure by design, and allows you to make informed decisions regarding the security level of a particular installation. Before using dummy trust or multiple hostnames, you should know that they may introduce serious security vulnerabilities if you are not carefully considering the security environment they are deployed in.

  • Added support for Configurable Trust for Programmatic SSL. Please see documentation for security caveats.
  • Added Support for Multiple Server Names. Please see documentation for security caveats.
  • Added Ability to Configure CASFilter via JNDI.
